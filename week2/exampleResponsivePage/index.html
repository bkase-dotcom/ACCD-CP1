One Story, Three Looks

This single page demonstrates how the same content can morph across devices. Resize your browser to see layout, color, type, and composition all shift at 699px and 1100px breakpoints—powered entirely by Flexbox.

Flex Layout

One Markup, Many Layouts

Flex containers drive header, hero, cards, and page structure. We only change directions, wrap, order, and sizing at breakpoints.

Media Queries

Three Breakpoints

Base (≤699px): dark, stacked. Tablet (≥700px): light, two-column editorial. Desktop (≥1100px): creamy studio with sticky aside.

Design Tokens

CSS Variables Flip Themes

Colors, radii, shadows, and type scale are CSS variables changed inside media queries—producing dramatic, cohesive shifts.

Composition

Reordering with Flex

The hero switches to side-by-side and even reverses order on desktop. Same elements, different composition.

Typography

System Sans → Serif → Sans

Mobile uses comfortable sans, tablet swaps to serif headings for an editorial vibe, desktop returns to confident sans.

Accessibility

Semantics Intact

Header, nav, main, and footer remain consistent. We don’t hide essential content—just restyle and rearrange it.

How It Works

Recipe

1) Mobile-first: Build content once, stacked. Use Flexbox for clusters (header, hero, cards).

2) Variables: Define colors, shadows, sizing via :root custom properties.

3) Media queries: At each breakpoint, update variables + flex properties (direction, wrap, gaps, order).

4) Test: Resize and inspect. Confirm the same DOM is expressed three ways.
